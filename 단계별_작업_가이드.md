# Git 커밋 작성자 수정 - 단계별 가이드

## 현재 문제
- "Your Name" 커밋 5개
- "MLeehwa" 커밋 24개  
- Vercel이 이들을 인식하지 못함

## 해결 방법 (3가지)

### 방법 1: Git Bash 사용 (가장 확실함) ⭐

**1단계: Git Bash 열기**
- 시작 메뉴에서 "Git Bash" 검색
- 또는 프로젝트 폴더(`C:\Users\LHA-M\WP1`)에서 우클릭 → "Git Bash Here"

**2단계: 프로젝트 폴더로 이동**
```bash
cd /c/Users/LHA-M/WP1
```

**3단계: 스크립트 실행**
```bash
bash fix-all-authors.sh
```
이 명령은 모든 커밋의 작성자를 "Mikepark0720"으로 변경합니다.

**4단계: 강제 푸시**
```bash
git push origin main --force
```

**5단계: Vercel에서 확인**
- Vercel 대시보드에서 "Create Deployment" 다시 시도
- 또는 GitHub에 푸시되면 자동 배포 확인

---

### 방법 2: Vercel 프로젝트 설정 확인

**1단계: Vercel 대시보드 접속**
- https://vercel.com/leehwas-projects/lhw-wp1/settings/git

**2단계: GitHub 저장소 연결 확인**
- "Disconnect" 클릭
- "Connect Git Repository" 클릭
- `Mikepark0720/LHSWP1` 선택
- 권한 확인

**3단계: 배포 재시도**
- Deployments 탭으로 이동
- "Create Deployment" 클릭
- Branch: `main` 선택
- "Deploy" 클릭

---

### 방법 3: 수동으로 명령 실행 (Git Bash에서)

Git Bash를 열고 다음 명령을 복사해서 실행:

```bash
cd /c/Users/LHA-M/WP1

git filter-branch -f --env-filter '
if [ "$GIT_AUTHOR_NAME" = "Your Name" ] || [ "$GIT_AUTHOR_NAME" = "MLeehwa" ]; then
    export GIT_AUTHOR_NAME="Mikepark0720"
    export GIT_AUTHOR_EMAIL="164405431+MLeehwa@users.noreply.github.com"
fi
if [ "$GIT_COMMITTER_NAME" = "Your Name" ] || [ "$GIT_COMMITTER_NAME" = "MLeehwa" ]; then
    export GIT_COMMITTER_NAME="Mikepark0720"
    export GIT_COMMITTER_EMAIL="164405431+MLeehwa@users.noreply.github.com"
fi
' --tag-name-filter cat -- --branches --tags

git push origin main --force
```

---

## 추천 순서

1. **먼저 방법 1 시도** (Git Bash + 스크립트)
2. 안 되면 **방법 2 시도** (Vercel 설정 확인)
3. 그래도 안 되면 **방법 3 시도** (수동 명령)

## 주의사항

⚠️ **중요**: 
- `git filter-branch`는 Git 히스토리를 재작성합니다
- 강제 푸시(`--force`)가 필요합니다
- 다른 사람과 협업 중이라면 사전에 공유하세요
- 작업 전에 백업을 권장합니다

## 확인 방법

작업 후 다음 명령으로 확인:
```bash
git log --format="%an|%ae" --all | findstr /V "Mikepark0720"
```
결과가 없으면 모든 커밋이 "Mikepark0720"으로 변경된 것입니다.
