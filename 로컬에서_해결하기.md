# 로컬에서 Vercel 배포 문제 해결하기

## 목표
- 기존 Vercel URL 유지
- 로컬에서 직접 배포 가능하게 하기

## 방법: Vercel CLI로 기존 프로젝트에 연결

### 1단계: 기존 프로젝트 정보 확인

Vercel 대시보드에서:
- https://vercel.com/leehwas-projects/lhw-wp1/settings/general
- "Project ID" 복사

### 2단계: 로컬에서 프로젝트 연결

터미널에서 실행:
```bash
vercel link
```

프롬프트에서:
- "Set up and deploy" 선택
- "Link to existing project" 선택
- 프로젝트 이름: `lhw-wp1` 선택
- 또는 Project ID 직접 입력

### 3단계: 직접 배포 (권한 우회)

```bash
vercel --prod --yes
```

이 방법은 Git 커밋 작성자 문제를 우회합니다.

---

## 대안: .vercel 폴더 수동 생성

### 1단계: Vercel 대시보드에서 정보 확인
- Project ID
- Org ID

### 2단계: .vercel 폴더 생성

`.vercel/project.json` 파일 생성:
```json
{
  "projectId": "여기에_Project_ID_입력",
  "orgId": "여기에_Org_ID_입력"
}
```

### 3단계: 배포
```bash
vercel --prod --yes
```

---

## 자동 배포 비활성화 (선택사항)

GitHub 연결은 유지하되, 자동 배포만 비활성화:
- Vercel 대시보드 → Settings → Git
- "Auto-deploy" 옵션 확인

그리고 필요할 때만 CLI로 배포:
```bash
vercel --prod --yes
```
