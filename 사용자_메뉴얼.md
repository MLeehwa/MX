## 시스템 개요

### WMS (Warehouse Management System)란?
이 시스템은 창고 관리 시스템(WMS)으로, 입고, 출고, 재고 관리, 위치 추적 등의 기능을 제공합니다.

### 주요 특징
- **웹 기반 시스템**: 브라우저에서 바로 사용 가능
- **역할 기반 접근**: Admin(관리자)와 PDA(현장 작업자) 역할 구분
- **다국어 지원**: 한국어, 영어, 스페인어 지원
- **실시간 데이터**: Supabase를 통한 실시간 데이터 동기화
- **바코드 스캔**: PDA에서 바코드 스캔을 통한 빠른 작업 처리

### 시스템 구조
```
WMS 시스템
├── 로그인 페이지 (login.html)
├── 관리자 대시보드 (admin/)
│   ├── 입고 계획
│   ├── 위치 보기
│   ├── 출고 확정
│   ├── 보고서
│   ├── 일일 입출고 리포트
│   └── 장소 마스터 관리
└── PDA 시스템 (pda/)
    ├── 입고 처리
    ├── 출고 처리
    ├── 위치 찾기
    └── 출하지시서 생성
```

## 시작하기

### 1. 로그인

1. 브라우저에서 시스템에 접속합니다.
2. 로그인 페이지가 자동으로 표시됩니다.
3. **Role 선택**: 
   - **Admin**: 관리자 권한 (모든 기능 접근)
   - **PDA**: 현장 작업자 권한 (PDA 기능만 접근)
4. **비밀번호 입력**: `12345` (기본 비밀번호)
5. **Login** 버튼을 클릭합니다.

> **참고**: ID는 선택한 Role과 자동으로 동일하게 설정됩니다.

### 2. 언어 변경

#### 관리자 페이지
- 상단 우측의 **한국어** 또는 **English** 버튼을 클릭하여 언어를 변경할 수 있습니다.

#### PDA 페이지
- 상단 우측의 **한국어**, **English**, **Español** 버튼을 클릭하여 언어를 변경할 수 있습니다.

### 3. 로그아웃

- 관리자: 사이드바 하단의 **로그아웃** 버튼 클릭
- PDA: 상단 우측의 **로그아웃** 버튼 클릭

## 관리자(Admin) 기능

관리자 대시보드에 접속하면 왼쪽 사이드바에 다음 메뉴들이 표시됩니다:

### 1. 📦 입고 계획 (Receiving Plan)

**기능**: 입고 예정 컨테이너를 등록하고 관리합니다.

**주요 작업**:
- 입고 예정 컨테이너 정보 입력
- 입고 날짜 설정
- 입고 계획 조회 및 수정
- Excel 파일을 통한 일괄 등록

**사용 방법**:
1. 사이드바에서 "📦 Receiving Plan" 클릭
2. 입고 계획 정보 입력
3. 저장 버튼 클릭

### 2. 📍 위치 보기 (Location View)

**기능**: 창고 내 제품/컨테이너의 위치를 시각적으로 확인합니다.

**주요 작업**:
- 위치 코드로 제품 검색
- 위치 맵에서 제품 위치 확인
- 위치별 재고 현황 조회

**사용 방법**:
1. 사이드바에서 "📍 Location View" 클릭
2. 위치 코드 또는 제품 정보로 검색
3. 맵에서 위치 확인

### 3. 🚚 출고 확정 (Shipping Confirmation)

**기능**: 출고 지시서를 확인하고 출고를 확정합니다.

**주요 작업**:
- 출고 지시서 목록 조회
- 출고 상태 확인
- 출고 확정 처리
- 출고 이력 조회

**사용 방법**:
1. 사이드바에서 "🚚 Shipping Confirmation" 클릭
2. 출고 지시서 목록에서 확인할 항목 선택
3. 출고 확정 버튼 클릭

### 4. 📊 보고서 (Report)

**기능**: 입출고 현황을 종합적으로 조회합니다.

**주요 작업**:
- 입고/출고 데이터 조회
- 기간별 통계 확인
- Excel 파일로 내보내기

**사용 방법**:
1. 사이드바에서 "📊 Report" 클릭
2. 조회 기간 설정
3. 데이터 조회 및 Excel 다운로드

### 5. 📈 일일 입출고 리포트 (Daily Report)

**기능**: 일일 입출고 현황을 한눈에 확인합니다.

**주요 작업**:
- 오늘 날짜의 입고/출고 현황 확인
- 입고/출고 통계 조회
- 차트를 통한 시각화

**사용 방법**:
1. 사이드바에서 "📈 일일 입출고 리포트" 클릭
2. 자동으로 오늘 날짜의 데이터가 표시됩니다
3. 차트와 통계를 확인합니다

### 6. ⚙️ 장소 마스터 관리 (Location Master)

**기능**: 창고 내 위치 정보를 관리합니다.

**주요 작업**:
- 위치 코드 등록/수정/삭제
- 위치 상태 관리 (사용 가능, 사용 중, 점검 중, 비활성화)
- 위치 좌표 설정 (x, y, width, height)
- 위치별 비고 입력

**사용 방법**:
1. 사이드바에서 "⚙️ 장소 마스터 관리" 클릭 (새 창에서 열림)
2. 위치 정보 입력 및 저장
3. 위치 목록에서 수정/삭제 가능

## PDA 기능

PDA 시스템은 현장에서 바코드 스캔을 통해 빠르게 작업을 처리할 수 있도록 설계되었습니다.

### 메인 화면

PDA 메인 화면에는 다음 정보가 표시됩니다:

- **입고 현황 카드**
  - 예정: 오늘 입고 예정된 컨테이너 수
  - 완료: 오늘 입고 완료된 컨테이너 수
  - 남은수: 아직 입고되지 않은 컨테이너 수

- **출고 현황 카드**
  - 예정: 오늘 출고 예정된 지시서 수
  - 완료: 오늘 출고 완료된 지시서 수
  - 남은수: 아직 출고되지 않은 지시서 수

### 1. 입고 처리 (Receiving)

**기능**: 바코드 스캔을 통해 입고를 등록합니다.

**사용 방법**:
1. 메인 화면에서 "입고 처리" 클릭
2. 컨테이너 바코드 또는 라벨 바코드 스캔
3. 입고 정보 확인 및 저장
4. 위치 코드 입력 (선택사항)

**주의사항**:
- 바코드 스캐너가 연결되어 있어야 합니다
- 스마트폰의 경우 카메라를 사용하여 바코드를 스캔할 수 있습니다

### 2. 출고 처리 (Shipping)

**기능**: 출하증 또는 제품 바코드를 스캔하여 출고를 확정합니다.

**사용 방법**:
1. 메인 화면에서 "출고 처리" 클릭
2. 출하증 바코드 또는 제품 바코드 스캔
3. 출고 정보 확인
4. 출고 확정 처리

### 3. 위치 찾기 (Location Finder)

**기능**: 제품이나 컨테이너의 위치를 맵에서 확인합니다.

**사용 방법**:
1. 메인 화면에서 "위치 찾기" 클릭
2. 제품 번호 또는 컨테이너 번호 입력/스캔
3. 맵에서 위치 확인

### 4. 출하지시서 생성 (Create Shipping Instruction)

**기능**: 바코드 스캔을 통해 출하지시서를 생성합니다.

**사용 방법**:
1. 메인 화면에서 "출하지시서 생성" 클릭
2. 제품 바코드 스캔
3. 출고 정보 입력
4. 출하지시서 생성

## 주요 기능 상세 설명

### 입고 프로세스

1. **입고 계획 등록** (Admin)
   - 관리자가 입고 예정 컨테이너 정보를 등록합니다
   - 입고 날짜, 컨테이너 번호, 제품 정보 등을 입력합니다

2. **입고 처리** (PDA)
   - 현장 작업자가 바코드를 스캔하여 실제 입고를 등록합니다
   - 입고 시간이 자동으로 기록됩니다

3. **입고 확인** (Admin)
   - 관리자는 입고 계획과 실제 입고 현황을 비교하여 확인할 수 있습니다

### 출고 프로세스

1. **출하지시서 생성** (PDA 또는 Admin)
   - 출고할 제품 정보를 입력하여 출하지시서를 생성합니다

2. **출고 처리** (PDA)
   - 현장 작업자가 바코드를 스캔하여 출고를 확정합니다
   - 출고 시간이 자동으로 기록됩니다

3. **출고 확정** (Admin)
   - 관리자는 출고 현황을 확인하고 필요시 수정할 수 있습니다

### 위치 관리

- **위치 코드**: 창고 내 특정 위치를 식별하는 고유 코드
- **위치 상태**:
  - `available`: 사용 가능
  - `occupied`: 사용 중
  - `maintenance`: 점검 중
  - `disabled`: 비활성화

## 데이터베이스 구조

시스템은 Supabase를 사용하며, 주요 테이블은 다음과 같습니다:

### 주요 테이블

1. **receiving_plan**: 입고 계획 정보
   - 컨테이너 번호, 입고 예정일, 제품 정보 등

2. **receiving_items**: 입고 품목 정보
   - 컨테이너 번호, 제품 번호, 수량, 위치 코드 등

3. **receiving_log**: 입고 로그
   - 실제 입고 시간, 라벨 ID 등

4. **shipping_instruction**: 출고 지시서
   - 출고 날짜, 상태, 제품 정보 등

5. **shipping_instruction_items**: 출고 지시서 품목
   - 출고된 라벨 ID, 출고 시간 등

6. **wp1_locations**: 위치 마스터
   - 위치 코드, 상태, 좌표 정보 등

## 문제 해결

### 로그인이 안 될 때
- 비밀번호가 `12345`인지 확인하세요
- 브라우저의 로컬 스토리지를 확인하세요
- 페이지를 새로고침하세요

### 데이터가 표시되지 않을 때
- 인터넷 연결을 확인하세요
- 브라우저 콘솔에서 오류 메시지를 확인하세요
- Supabase 연결 상태를 확인하세요

### 바코드 스캔이 안 될 때
- 바코드 스캐너가 제대로 연결되어 있는지 확인하세요
- 스마트폰의 경우 카메라 권한을 확인하세요
- 바코드가 손상되지 않았는지 확인하세요

### 위치 정보가 표시되지 않을 때
- 위치 마스터에 해당 위치가 등록되어 있는지 확인하세요
- 위치 코드 형식이 올바른지 확인하세요
